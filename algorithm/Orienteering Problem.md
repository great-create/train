# 演算法作業：限時獎勵收集配送（Orienteering Problem）

請對**同一問題**分別用四種方法各實作：**動態規劃、貪婪、回溯、分枝界限**。

## 問題定義

一台車於時刻 0 從倉庫出發，須在 `T_max` 前回到倉庫。共 N 個節點 `0 ~ N-1`，節點 0 為倉庫（無獎勵）。每個客戶 i 有獎勵 `p_i` 與服務時間 `s_i`（抵達後完成服務所需的時間）。行駛時間由 N×N 矩陣 `t` 給定，`t[i][j]` 為 i→j 的時間（**可不對稱**）；當 i≠j 且 `t[i][j]=0` 表無路。車輛**可略過客戶**，總花費時間（行駛 + 服務）不得超過 `T_max`。

**目標：在 `T_max` 時間限制下，最大化被服務客戶的獎勵總和。**（本問題為 NP-hard）

## 輸入

```
N T_max
p_i s_i              ← N 行，依序為節點 0..N-1
t[i][j]              ← N 行，每行 N 個整數
```

## 輸出

```
最大獎勵總和
0 a b ... 0
```

路徑為以**空白分隔**的節點編號。合法路徑須滿足：頭尾皆為倉庫 0；總花費時間 ≤ `T_max`；相鄰兩點的 `t` 值非 0；不重複拜訪；宣告獎勵 = 實際被訪客戶獎勵總和。若一位都無法服務，輸出 `0` 與 `0 0`。
